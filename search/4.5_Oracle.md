# 오라클 (Oracle)

## 1. 암호학에서 '오라클(Oracle)'이란
- 오라클 (Oracle): 입력을 주면 정해진 규칙에 따라 즉각적으로 올바른 출력을 돌려주는 가상의 Black-box
- 특징: 내부 동작의 은닉성 (Black-box): 사용자는 오라클 내부에서 어떤 비밀키가 사용되는지, 구체적으로 어떤 연산을 거치는지 알 수 없으며 오직 입력(Query)과 출력(Response)만을 관찰할 수 있다.
- 역할:
  - 공격자 관점: 공격 모델에서 공격자가 가질 수 있는 극대화된 능력을 수학적으로 묘사하는 도구.
  - 방어자(증명자) 관점: 암호 프로토콜의 안전성을 증명할 때, 실제 구현하기 복잡한 요소를 이상적인 수학적 대상으로 단순화하여 엄밀한 증명을 가능하게 돕는 도구.
 
- 4_DS에선 두 종류의 오라클이 나왔다.
  - 서명 오라클 (Signing Oracle): 공격 모델(UF-CMA)에서 공격자의 능력을 묘사하기 위해 사용됨.
  - 무작위 오라클 (Random Oracle): 해시 함수의 이상적인 형태를 가정하여 수학적 안전성 증명(Reduction)을 수행하기 위해 사용됨.

## 2. 서명 오라클 (Signing Oracle)
- 서명 오라클 (Signing Oracle): 공격자가 비밀키($sk$)를 알지 못하는 상태에서, 원하는 임의의 메시지 $M$을 입력하면 그에 대한 올바른 서명값 $\sigma \leftarrow \text{S}_{sk}(M)$을 돌려주는 주체이다.

- UF-CMA의 철학: 왜 이런 강력한 오라클을 공격자에게 부여할까?
  - 실제 환경에서 해커는 합법적인 서명자가 과거에 서명했던 수많은 데이터(이메일, 전자상거래 계약서 등)를 가로채거나, 합법적인 서명자를 속여서 특정 문서에 서명하도록 유도할 수 있다.
  - 따라서 현대 암호학에서는 "해커가 당신에게 원하는 모든 메시지의 서명을 받아낼 수 있는 오라클(서명 오라클)을 가지고 있더라도, 단 한 번도 서명받은 적 없는 새로운  메시지 $M$^에 대한 위조 서명 $\sigma$^를 만들어내는 것은 불가능해야 안전하다"고 정의한다.
  - 이것이 UF-CMA(Chosen Message Attack 하에서의 위조 불가능성)이다.
 
- 오라클의 작동 프로세스
  - Initialize 단계: 방어자(시스템)가 키 쌍 $(vk,sk)$를 생성하고 공개키 $vk$만 공격자에게 준다.
  - Sign 단계 (오라클 쿼리): 공격자는 원하는 메시지들을 서명 오라클에 보낸다. 오라클은 비밀키 $sk$를 사용해 서명을 생성하여 돌려주고, 시스템은 공격자가 요청한 메시지 목록 $S$를 기록한다.
  - Finalize 단계: 공격자가 위조라고 주장하는 쌍 $(M, \sigma)$를 제출한다. 이때 승리하려면 다음 두 조건을 만족해야한다.
    - $V_{vk}(M, \sigma) = 1$ // 서명이 검증을 통과해야 함
    - $M \notin S$ // 서명 오라클에 물어본 적 없는 새로운 메시지여야 함

- Plain RSA에서 서명 오라클 악용한 공격
  - Plain RSA는 해시 함수 없이 곱셈 준동형성 $(x^d \mod N)$이 그대로 노출되어 서명 오라클을 통해 쉽게 깨진다.
    - 1. 공격자가 위조하고자 하는 타깃 메시지가 $M_3 = y_1 \cdot y_2 \mod N$이면,
      2. 서명 오라클에 각각 $y_1$과 $y_2$를 쿼리하여 합법적인 서명 $x_1 = y_1^d \mod N$과 $x_2 = y_2^d \mod N$을 얻는다
      3. 이를 곱하여 오라클 도움 없이 $x_3 = x_1 \cdot x_2 \mod N$을 계산하면, 이는 $M_3$에 대한 완벽한 위조 서명이 된다.
    - $\therefore$ 공격자가 서명 오라클에 단 두 번만 질의하면, 질의한 적 없는 제3의 메시지 $(y_1 \cdot y_2)$에 대한 완벽한 위조 서명을 만들어 낼 수 있으므로 UF-CMA 안전성을 전혀 만족하지 못한다.

## 3. 무작위 오라클 (Random Oracle)
- 실제 암호학적 해시 함수(SHA-2, SHA-3 등)는 수학적으로 고도로 설계되었지만 복잡한 비트 연산의 덩어리이므로, 이 해시 함수를 포함한 전체 서명 스킴의 안전성을 수학적으로 증명하는 것은 극도로 어렵다.
- 이를 해결하기 위해 도입된 패러다임이 무작위 오라클 모델(ROM)이다.

- 개념: 해시 함수 $H$를 대변하는 이상적인 가상의 오라클
- 동작:
  - 1. 임의의 입력 $M$이 들어왔을 때, 이전에 들어온 적이 없는 새로운 입력이라면 출력 공간에서 완전한 무작위 값을 하나 뽑아서 반환하고, 이 값을 내부 테이블에 기록해 둔다.
    2. 만약 이전에 물어봤던 동일한 입력 $M$이 다시 들어오면, 테이블을 조회하여 과거에 뱉었던 것과 똑같은 값을 돌려준다.
  - 일관성(Consistency)과 무작위성(Randomness)을 동시에 완벽하게 갖춘 함수 모델이다.

- 무작위 오라클을 쓰는 이유
  - PKCS#1의 한계: 고정된 패딩 바이트 때문에 입력값들이 일방향성(One-wayness)으로 논리적 환원(Security Reduction)이 불가능했다.
  - FDH의 해결책: 해시 함수를 통해 결과값이 모듈러 공간 전체에 고르게 퍼지도록 확장(Full Domain Hash)했다. 해시 함수를 무작위 오라클로 모델링하면 해시의 출력이 완전히 균등 분포(Uniform Distribute)를 따르므로, 수학적 증명이 가능하다.

## 4. Random Oracle Model에서의 FDH 안전성과 수학적 손실의 의미
$$Adv_{FDH}ᵘᶠ⁻ᶜᵐᵃ(A) ≤ q_H · Adv_{RSA}ᵒʷ(B)$$
- q_H: 공격자가 무작위 오라클(해시 함수 $H$)에 던진 쿼리의 최대 횟수 // 보통 실무적으로 $2^{60}$에서 $2^{80}$번까지 가능하다고 가정
- Adv_{RSA}ᵒʷ(B): RSA Trapdoor Permutation의 일방향성(역원 찾기) 문제 해결 확률

- 안전성 환원(Security Reduction)과 오라클 시뮬레이션
  - 이 수식의 증명은 "만약 FDH 서명을 위조할 수 있는 공격자 $A$가 존재한다면, 이를 서브루틴으로 사용하여 난해한 수학 문제인 RSA 일방향성 문제를 푸는 알고리즘 B(시뮬레이터)를 만들 수 있다"는 논리로 진행된다.
